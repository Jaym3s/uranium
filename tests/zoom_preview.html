Zoom Preview

How To:

This is a widget to look at a large zoomed image with touch events (although I've simulated it with mouseevents). The idea is that given some thumbnails / image preview, you can insert a button that the user can touch/drag around to view the zoomed image. It seems in the wild we also need a modifier rule, which tells us how to generate the src of the big image so that its really giant.

Note: We assume listeners on the thumbnails to update the main image are already in place. If this is not the case, we can make a skeleton image gallery widget.

To setup zoom preview:

  1. Group the zoom-preview. You can do this with either the structural method or the unique ID method (add link).
  2. Identify the zoom-container. This should contain the normal 'hero' image that the thumbnails update. It should also contain the zoom-button and zoom-image elements.
    • Set the 'data-ur-zoom-preview-component' to 'container' on the zoom-container.
    • Note : Its ok for the data-ur-set and zoom-container to be the same element. This will probably be the case when you don't have to deal with thumbnails.
  3. Set the 'data-ur-zoom-preview-component' attribute to 'zoom_image' on the zoom image element (which should be a descendant of the zoom-container).
      You can also specify a modifier rule for the zoom-image:
    • Normally, when a thumbnail is clicked, its src attribute is copied to the zoom image -- but this usually results in an image thats much too small to zoom.
    • Set the 'data-ur-src-modifier-match' attribute's value to be the match regex to be applied to the thumbnail image's src
    • Set the 'data-ur-src-modifier-replace' attribute's value to be the replace string to be applied to the thumbnail image's src
    • Look at this page's html for an example of this
  4. Set the 'data-ur-zoom-preview-component' attribute to 'button' the button element (which should be a descendant of the zoom-container). The 'loaded' class is added to this element once the image has loaded.
      (Optional - but highly recommended) : Add a modifier rule for the image displayed on the button. (Its good to modify the url so that a giant image isn't displayed in a tiny space. That results in slow re-paints which makes the UX janky). This modifier gets applied to the zoom-image url (after the zoom-image modifier gets applied):
    • Set the 'data-ur-src-modifier-match' attribute's value to be the match regex for the zoom-image src
    • Set the 'data-ur-src-modifier-replace' attribute's value to be the replace string for the zoom-image src
  5. Set the 'data-ur-zoom-preview-component' attribute to 'normal_image' on the normal image. (The normal image must be a descendant of the zoom-container).
      Add a match/replace pair. These get applied to the normal image src to generate the initial zoom-image url:
    • Set the 'data-ur-zoom-modifier-match' attribute.
    • Set the 'data-ur-zoom-modifier-replace' attribute
  6. (Optional) Identify the zoom-thumbnails-container. This should contain the all of the thumbnail images. Set the 'data-ur-zoom-preview-component' to 'thumbnails' the zoom-thumbnails-container
